# Deploy Netrik FastAPI to Render

## Prerequisites

Your repo must include the trained model artifacts and data files. The API needs:

```
artifacts/
  tabular/sklearn_xgb_pipeline.joblib
  sequence/lstm_encoder.pt
  fusion/fusion_mlp.pt
  graph/precomputed_node_embeddings.npy
  graph/node_embedding_index.pkl
data/
  raw/TRAIN.csv
  processed/sequence_features.npy
  processed/sequence_ids.npy
  graph/node_ids.npy
```

Run your training pipelines first if these are missing, then commit them.

## Deploy Steps

1. **Push to GitHub** – Ensure your code (including `artifacts/` and `data/`) is in a GitHub repo.

2. **Connect to Render** – Go to [render.com](https://render.com) → Sign up/Login → Dashboard.

3. **New Blueprint** – Click **New** → **Blueprint**. Connect your GitHub repo.

4. **Deploy** – Render will detect `render.yaml` and create:
   - A free PostgreSQL database (`netrik-db`)
   - A web service (`netrik-api`) with the FastAPI app

5. **Wait for build** – The first build may take 10–15 minutes due to PyTorch. Render will:
   - Install dependencies from `requirements.txt`
   - Run `uvicorn src.serving.app:app --host 0.0.0.0 --port $PORT`
   - Use `/health` for health checks

6. **Get your URL** – After deployment, your API will be at `https://netrik-api.onrender.com` (or the name Render assigns).

## Environment Variables (auto-configured)

| Variable      | Source                                                |
|---------------|-------------------------------------------------------|
| `DATABASE_URL`| From `netrik-db` (PostgreSQL)                         |
| `JWT_SECRET`  | Auto-generated by Render                              |
| `PORT`        | Set by Render                                         |

## API Endpoints

- `GET /health` – Health check
- `POST /login` – Get JWT token
- `POST /score` – Score applicant (requires `Authorization: Bearer <token>`)
- `POST /explain` – Explain decision (admin only)

## Notes

- **Free tier**: Services spin down after inactivity; first request may be slow.
- **Build time**: PyTorch CPU build can take ~10–15 minutes.
- **Database**: Tables are created automatically on first request via `init_db()`.
